# Start Here

Ce projet fait partie d'un parcours de 4 mois.
Le but n'est pas de tout faire d'un coup, mais d'appliquer les competences du cours dans le bon ordre.

## Ordre du cours a respecter

1. Frontend simple: HTML -> CSS -> JavaScript
2. Frontend moderne: React avec Vite -> Tailwind CSS
3. Backend: Node.js -> Express -> MongoDB
4. Integration finale: connecter le frontend au backend

## Ce que vous devez faire avec ce dossier

Utiliser les fichiers de `resources/` comme support pour reconstruire le site pas a pas.

## Par quoi commencer

1. Lire [README.md](./README.md)
2. Ouvrir [ui-guide/ui-guide.html](./ui-guide/ui-guide.html)
3. Lire [CHECKLIST.md](./CHECKLIST.md)
4. Garder [COMMON-MISTAKES.md](./COMMON-MISTAKES.md) a cote pendant le travail

## Comment avancer

### Phase 1. HTML + CSS

- creer la structure des pages
- faire la navbar, le footer, les sections principales
- utiliser les textes de `content/pages/`
- utiliser les regles de `brand-system/`

### Phase 2. JavaScript

- ajouter les interactions simples
- menu mobile
- panier local
- carousel simple
- validation visuelle du formulaire

### Phase 3. React + Vite + Tailwind

- transformer les pages en composants
- organiser le projet proprement
- reutiliser les sections communes
- appliquer le style final avec Tailwind

### Phase 4. Backend

- creer les routes produits
- preparer MongoDB avec les donnees de `seed-data/products.fr.json`
- exposer les donnees necessaires au frontend

### Phase 5. Connexion frontend/backend

- recuperer les produits depuis l'API
- afficher les details produit
- connecter le panier et le checkout avec les donnees finales

## Conseil important

Ne sautez pas directement a MongoDB ou a la logique backend si l'interface n'est pas encore construite.
Sur ce projet, il vaut mieux:

1. voir les pages correctement
2. rendre les interactions simples
3. brancher les vraies donnees ensuite
